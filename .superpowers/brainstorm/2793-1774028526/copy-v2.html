Before / After — Take 2

피드백 반영. "프로세스가 반복 실패 + 재시작" 하는 cascade와, 양쪽 모두의 비싼 시간 낭비가 느껴지도록.

Before Side

v1에서 빠진 것: (1) chart 뒤져서 의사 찾기 → 연락 → 안 됨 → 다시 원점 → option 2, 3 반복루프. (2) 병원쪽도 변경 (시간, 캔슬) → 또 재시작. (3) coordinator만의 문제가 아님 — 의사, 병원 양쪽의 비싼 시간도 같이 burn. (4) 바빠서 서류 실수 → 재정 손실 → 정확도 올리려면 더 많은 시간 필요 → 악순환.
A The Loop — 반복되는 실패 사이클

Picking the right doctor for a surgery?

5 minutes.

Actually getting them assigned?

Search a spreadsheet for who's available. Call them. Schedule conflict — they didn't update their calendar.
Back to the chart. Try the next doctor. Call again.
Meanwhile the hospital changes the surgery time. Start over.

It's not just the coordinator's time. Doctors wait. Hospitals wait. Everyone's expensive hours, stuck in a loop.

Multiply that by

20+ cases a day

가장 서사적. 실패 → 재시작 루프를 글로 보여줌. "모두의 시간" 강조. 숫자는 마지막 punch.

B Cascade — 짧고 구조적

The decision takes 5 minutes. The process takes all day.

1

Search a chart for available doctors. Call one. Can't do it — outdated schedule.

2

Try the next. Call again. Meanwhile, hospital changes the time. Start over.

3

Repeat across 20+ cases a day. Doctors wait. Hospitals wait. Everyone's billable hours, burning.

And when rushed coordinators make a paperwork mistake? Financial errors. Fix those? Even more hours.

번호 매긴 cascade. 1→2→3 읽는 흐름이 자연스러움. 마지막에 재정 리스크까지 커버. Pitch deck에도 바로 쓸 수 있는 구조.

C Headline Punch — 한 줄 + 보조

A 5-minute decision. An all-day process.

For every surgery, a coordinator searches spreadsheets, calls doctors, hits dead ends, and starts over. Schedules change mid-call. Doctors don't pick up. Hospitals cancel.

It's not just one person's problem. Doctors, hospitals, coordinators — all three burn hours waiting on a process built for one case at a time.

Rushed work means paperwork errors. Fixing errors means more hours, more cost.

Headline 한 줄이 모든 걸 요약 ("5-min decision, all-day process"). 아래는 보충. 가장 기억에 남는 한 줄.

After Side

매칭 원칙: Before의 각 pain이 After에서 1:1로 뒤집혀야 함.
Loop → eliminated. Everyone's time → recovered. Errors → prevented.
A Pain → Fix 1:1 Mapping

The loop, eliminated.

Search a chart for availability  →  See it. Drag. Drop. Assigned.

Call, fail, try the next one  →  Real-time availability. No dead ends.

Hospital changes, start over  →  Changes sync instantly. Everyone sees it.

Paperwork mistakes, financial errors  →  One system, one record. Audit trail built in.

Three portals — hospital, coordinator, doctor — on one shared platform.

Before의 pain 하나하나를 취소선으로 지우면서 솔루션 제시. 가장 논리적이고 설득력 있음. A→B 즉각 비교.

B Time Payoff + Pain Answers

The all-day process?

Now it's the 5-minute decision.

Phone calls and spreadsheets? Replaced. Drag & drop with live availability.

Dead ends and do-overs? Gone. One shared state, always current.

Paperwork errors? Prevented. Single source of truth, full audit trail.

One platform for hospitals, coordinators, and doctors.

Headline이 Before를 직접 뒤집음 ("all-day process → 5-min decision"). Pain → one-word answer → brief explanation. Before C (headline punch)와 짝이 잘 맞음.

C Rhythm Statement

One platform. One truth.
Zero phone calls. Zero do-overs.

See every doctor's real-time availability. Assign in seconds.

Changes sync across all three parties, instantly.

Every action tracked. Every dollar accounted for.

"One, One, Zero, Zero" 리듬. 기억에 남지만 Before의 pain과 직접 매핑은 약함. 더 브랜드/마케팅 느낌.
